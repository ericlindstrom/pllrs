PLLRS

Responsive CSS Column Structure

What is it?

PLLRS is a library for responsive uniform column structure. Built considering mobile first, It handles 4 breakpoints with media queries. By default, they are:

  • base (mobile first)
  • 600px (small tablet)
  • 768px (large tablet)
  • 1000px (desktop)

How Do I use it?

It's simple.

<div class="pllrs4 _3_2">
  <div class="column"></div>
  <div class="column"></div>
  ...
</div>

You start with pllrs plus the number of columns you want for a desktop site, in this case 4 (pllrs4). without a secondary class (_3_2), this will default to 4 Columns, until it hits mobile, which will then be a single column (4441).

What do the underscores mean?

pllrs uses a secondary class to inform the columns where to "fold". If a second class (beginning with underscores) is not found, it will default to XXX1. Otherwise, the underscore will indicate a "hold". If you wish to hold all the way from desktop to mobile use 4 underscores (pllrs3 ____).

How big is it?

8.1kb minified and gzipped.

Does it support nesting?

You betcha.

Do I need to use divs?

Nope, feel free to use whatever HTML element you want.

What browsers does it support?

pllrs works with all browsers that support CSS3 selectors and media queries. If you need support for browsers below Internet Explorer 9, you'll want to add respond.js and selectivizr. These have not been tested, but will be in the future.

Can I change the breakpoints?

You sure can. pllrs was built with percentages to allow for fluid scaling. These can be changed in the pllrs/pllrs.scss file.
